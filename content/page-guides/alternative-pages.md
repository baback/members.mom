# Writing Guide: Alternative Pages (`/alternative/[slug]`)

## Overview

These are bottom-of-funnel pages targeting people actively searching for alternatives to a specific competitor. The reader already knows the competitor, is frustrated or curious, and is comparing options. Your job is to be the most helpful, honest comparison page on the internet — not a sales pitch.

## Tone & Voice

- Write like a knowledgeable friend who's tested all these tools, not a marketer
- Be opinionated. Say what's good AND what's bad about each tool, including ReUGC
- Use "you" and "your" — speak directly to the reader
- No hype words (revolutionary, game-changing, incredible). State facts and let the reader decide
- Conversational but efficient. No filler paragraphs
- Every sentence should either inform or help the reader make a decision

## Total Word Count

**2,500–3,500 words** per page. Long enough to be comprehensive, short enough to stay useful.

## Section-by-Section Breakdown

### 1. H1 + Meta Title
- Format: "Best [Competitor] Alternative for AI UGC Videos (2026)"
- Meta description (155 chars max): Include competitor name, mention "5 alternatives compared", include a differentiator like "with pricing"
- Example meta: "Looking for a [Competitor] alternative? We compared 6 AI UGC tools on price, quality, and features. Here's what actually works in 2026."

### 2. Opening Paragraph (80–120 words)
- Start with WHY someone would leave [competitor]. Be specific — don't just say "it's expensive"
- Name 2–3 concrete pain points (e.g., "limited avatar selection", "no batch generation", "$300/month for 10 videos")
- End with what this page covers: "We tested 6 alternatives head-to-head. Here's what we found."
- **Do NOT start with "In today's world..." or "Are you looking for..." — start with the pain**

### 3. Quick Verdict Box (50–80 words)
- A highlighted/bordered box at the top for skimmers
- Format: "**TL;DR:** If you're leaving [Competitor] because of [main reason], [Tool A] is the best pick for [use case]. If you need [other thing], go with [Tool B]. For the best price-to-quality ratio, ReUGC gives you [specific benefit]."
- This box should be genuinely helpful, not just "pick us"

### 4. What to Look For in a [Competitor] Alternative (150–200 words)
- 4–5 bullet points of evaluation criteria
- Each bullet: criteria name in bold + 1 sentence explaining why it matters
- Example: "**AI Actor Variety** — If you're running ads across demographics, you need diverse actors. Some tools have 5, others have 100+."
- This section builds trust — shows you have a methodology, not just opinions

### 5. #1 Pick: ReUGC (300–400 words)
- Hero-style card with clear visual hierarchy
- **What it is** (1 sentence)
- **Why it's #1** (3–4 bullet points with specifics — pricing numbers, feature counts, concrete benefits)
- **Best for** (1 sentence — who should pick this)
- **Pricing** (exact tier, exact price — e.g., "Starts at $49/month for 10 videos")
- **Limitations** (1–2 honest limitations — e.g., "No free tier", "Newer platform, smaller community")
- The limitations section is CRITICAL. It builds trust. If you only say good things, readers bounce.

### 6. Alternatives #2–#5 (200–300 words each, 4 tools)
- Each tool gets its own card/section
- Structure per tool:
  - **Tool name + 1-line description**
  - **Best for:** (1 sentence)
  - **Pros:** (3–4 bullets, be specific — "150+ avatar options" not "lots of avatars")
  - **Cons:** (2–3 bullets, be honest — "Starts at $89/month", "No UGC-style output")
  - **Pricing:** (exact numbers)
  - **Verdict:** (2 sentences — when to pick this tool over others)
- Pick tools that are genuinely relevant to the competitor being replaced. Don't pad with irrelevant tools.

### 7. Feature Comparison Table
- Full-width table, scannable
- Columns: Tool Name | Price/mo | Videos/mo | AI Actors | Languages | UGC Style | Batch Creation | API
- Use checkmarks (✓), crosses (✗), and specific numbers — never "Yes/No" when you can be specific
- ReUGC row should be highlighted but not obnoxiously
- Include the competitor being replaced as the first row so readers can compare directly

### 8. How We Evaluated (100–150 words)
- Brief methodology section
- "We signed up for each tool, created the same test video (a 30-second product testimonial), and compared output quality, generation speed, pricing, and feature depth."
- This section exists for E-E-A-T (Google's experience/expertise signals). Keep it short but credible.

### 9. FAQ (3–4 questions, 60–80 words each answer)
- Questions should be things people actually search:
  - "Is [Competitor] worth it in 2026?"
  - "What's the cheapest [Competitor] alternative?"
  - "Can AI UGC really replace human creators?"
  - "Does [Competitor] offer a free plan?"
- Answers should be direct. First sentence = the answer. Then 1–2 sentences of context.
- Use FAQ schema markup for rich snippets

### 10. CTA Section
- Use the standard site CTA component (dark bg, headline, video fan)
- Copy: "Stop overpaying for [competitor's weakness]. Start scaling with AI."
- Button: "Get Started"

## Formatting Rules

- Use H2 for main sections, H3 for individual tool names
- Bold tool names on first mention in each section
- Use bullet points for pros/cons — never paragraphs
- Comparison table must be responsive (horizontal scroll on mobile)
- Every tool mention should link to its own alternative page if one exists (internal linking)
- Add a "Last updated: [month] 2026" line below the H1 for freshness signals

## What NOT to Do

- Don't write generic filler like "In the world of AI video, there are many options..."
- Don't be a shill — if a competitor is genuinely better at something, say so
- Don't use the same pros/cons for every tool — each should feel individually researched
- Don't skip pricing. Readers on alternative pages ALWAYS want to know the cost
- Don't write 6 paragraphs of intro before getting to the tools. Respect the reader's time
- Don't use stock phrases like "without further ado" or "let's dive in"
